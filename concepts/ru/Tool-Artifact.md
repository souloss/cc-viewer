# Artifact

Визуализируйте HTML или Markdown файл как артефакт — веб-страницу с приватностью по умолчанию, размещённую на claude.ai, которую пользователь может открыть в браузере и позже выбрать для совместного использования. Используйте её, когда визуальное представление лучше текста в терминале.

## Когда использовать

- Публикация визуального результата: отчёта, панели мониторинга, анализа ошибок или макета интерфейса
- Обновление ранее опубликованной страницы на месте (один и тот же путь файла повторно развёртывается на один и тот же URL)
- Получение списка существующих артефактов пользователя для поиска одного из предыдущей сессии (`action: "list"`)
- **Не** для контента, который должен оставаться локальным, простых текстовых ответов или чего-либо, требующего внешних сетевых ресурсов во время просмотра — строгая CSP блокирует каждый внешний хост

## Параметры

- `file_path` (строка): Путь к файлу `.html` или `.md` для визуализации. Файл оборачивается в структуру документа при публикации, поэтому пишите содержимое страницы непосредственно — без тегов `<!DOCTYPE>`, `<html>`, `<head>` или `<body>`. Один и тот же путь → один и тот же URL при повторном развёртывании; другой путь получает новый URL.
- `favicon` (строка, требуется для публикации): Один или два эмодзи, используемые как значок вкладки браузера (например, `"📊"`). Только эмодзи, без разметки. Оставляйте его одинаковым при повторных развёртываниях — пользователи находят вкладку по иконке.
- `description` (строка): Однострочное описание, показываемое на карточке галереи артефактов.
- `url` (строка, необязательный): Передайте URL существующего артефакта для обновления его на месте из разговора, который его не публиковал. Без неё новый разговор всегда создаёт новый URL.
- `label` (строка, необязательный): Краткое имя версии (макс. 60 символов), показываемое в выборщике версий.
- `action` (строка, необязательный): `"publish"` (по умолчанию) или `"list"` — список опубликованных артефактов пользователя (название, URL, время последнего обновления), опционально с `limit`.
- `force` (логический, необязательный): Перезапись без проверки конфликтов. Только после получения 409 от одновременной записи, после согласования.

## Примечания

- **Только автономный контент.** Строгая CSP блокирует запросы на любые внешние хосты — CDN-скрипты, внешние таблицы стилей, удалённые изображения, fetch/WebSockets. Встраивайте весь CSS/JS и встраивайте ассеты как URI `data:`.
- **Адаптивный и поддерживающий тему.** Страницы визуализируются в светлой или тёмной теме зрителя; оформляйте обе (`prefers-color-scheme` плюс переопределение `data-theme` зрителя). Широкий контент прокручивается в собственном контейнере — тело страницы никогда не должно прокручиваться горизонтально.
- **Обновление в разговорах требует `url`.** Повторное развёртывание одного и того же пути файла повторно использует URL только в разговоре, который его опубликовал; чтобы сохранить ссылку на старый артефакт, найдите его URL с помощью `action: "list"` и передайте как `url`.
- **Публикация является внешней операцией.** Контент, отправленный на сервис артефактов, может кэшироваться даже при последующем удалении — не публикуйте ничего, что должно оставаться приватным на машине.
- **Прочитайте обратно с WebFetch.** URL артефактов claude.ai получаемы через WebFetch (не curl, который получает оболочку приложения).
