# ReportFindings

Сообщайте выводы проверки кода как типизированный, структурированный список, который хост-интерфейс визуализирует нативно — вместо печати как текста в чате.

## Когда использовать

- Завершение проверки кода, чьи активные инструкции явно говорят сообщать выводы с помощью этого инструмента
- Повторное сообщение после применения исправлений, когда инструкции проверки просят это (каждый вывод затем содержит `outcome`)
- **Не** для случайных мнений, обычных ответов или проверок, чьи инструкции указывают другой формат вывода — и никогда рядом с текстовым дубликатом тех же выводов

## Параметры

- `findings` (массив, обязательный, макс. 32): Проверенные выводы, ранжированные по серьёзности вверх — пустой массив, если ничего не выжило проверку. Каждый вывод:
  - `file` (строка, обязательная): Путь относительно репозитория.
  - `line` (число, необязательное): Якорная строка, индексированная с 1.
  - `summary` (строка, обязательная): Однострочное утверждение о дефекте.
  - `failure_scenario` (строка, обязательная): Конкретные входные данные/состояние → неправильный выход или крах.
  - `category` (строка, необязательная): Краткий слаг в формате kebab-case, например `correctness`, `simplification`, `efficiency`, `test-coverage`.
  - `verdict` (строка, необязательная): `CONFIRMED` или `PLAUSIBLE` — устанавливается, когда прошла проверка; отсутствует при встроенных проверках.
  - `outcome` (строка, необязательная): ТОЛЬКО при повторном сообщении после исправлений — `fixed`, `skipped` или `no_change_needed`.
- `level` (строка, необязательная): Уровень усилий, на котором работала проверка — `low`, `medium`, `high`, `xhigh` или `max`.

## Примечания

- **Вызовите один раз.** Один вызов с полным, проверенным, отсортированным по серьёзности списком — не по одному вызову на вывод.
- **Пусто — действительный результат.** Если ничего не выжило проверку, сообщите пустой массив, а не заполняйте слабыми выводами.
- **Не дублируйте в тексте.** Когда этот инструмент сообщает результаты, выводы не должны быть также напечатаны как сообщение в чате.
- **`outcome` только для повторных сообщений.** При первом сообщении оставьте его не установленным; после прохождения применения установите, что действительно произошло с каждым выводом.
