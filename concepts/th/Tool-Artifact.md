# Artifact

แสดงไฟล์ HTML หรือ Markdown เป็น Artifact — หน้าเว็บที่โฮสต์บน claude.ai ซึ่งเป็นแบบส่วนตัวตามค่าเริ่มต้น และผู้ใช้สามารถเปิดในเบราว์เซอร์และเลือกแชร์ในภายหลัง ใช้เมื่อการสื่อสารทางภาพดีกว่าข้อความ terminal

## เมื่อใดควรใช้

- การเผยแพร่ผลงานทางภาพ: รายงาน, แดชบอร์ด, การสอบสวนข้อบug, หรือ UI mockup
- การอัปเดตหน้าที่เผยแพร่ก่อนหน้านี้ (redeploy ที่พาธเดียวกัน → URL เดียวกัน)
- การแสดงรายการ artifacts ที่มีอยู่ของผู้ใช้เพื่อค้นหาจากเซสชันก่อนหน้า (`action: "list"`)
- **ไม่ใช้** สำหรับเนื้อหาที่ต้องอยู่ในเครื่อง, คำตอบเป็นข้อความปกติ, หรือสิ่งใดก็ตามที่ต้องการทรัพยากรเครือข่ายภายนอกในเวลาแสดง — CSP ที่เข้มงวดบล็อกทุกโฮสต์ภายนอก

## พารามิเตอร์

- `file_path` (สตริง): พาธไปยังไฟล์ `.html` หรือ `.md` ที่จะแสดง ไฟล์ถูกห่อหุ้มด้วยโครงกระดูกเอกสารในเวลาเผยแพร่ ดังนั้นเขียนเนื้อหาหน้าโดยตรง — ไม่มีแท็ก `<!DOCTYPE>`, `<html>`, `<head>`, หรือ `<body>` พาธเดียวกัน → URL เดียวกันบน redeploy; พาธที่แตกต่างกันอ้างถึง URL ใหม่
- `favicon` (สตริง, จำเป็นต่อการเผยแพร่): emoji หนึ่งหรือสองตัวใช้เป็นไอคอนแท็บเบราว์เซอร์ (เช่น `"📊"`). emoji เท่านั้น, ไม่มี markup. เก็บเหมือนเดิมในแต่ละ redeploy — ผู้ใช้หาแท็บของตนโดยไอคอน
- `description` (สตริง): ซับไตเติลประโยคเดียวที่แสดงบนการ์ดแกลเลอรี artifact
- `url` (สตริง, ไม่จำเป็น): ส่ง URL ของ artifact ที่มีอยู่เพื่ออัปเดตในที่จากเซสชันที่ไม่ได้เผยแพร่มันไว้ โดยไม่มีสิ่งนี้ เซสชันใหม่จะสร้าง URL ใหม่เสมอ
- `label` (สตริง, ไม่จำเป็น): ชื่อเวอร์ชันสั้นและอ่านได้ (สูงสุด 60 ตัวอักษร) ที่แสดงในตัวเลือกเวอร์ชัน
- `action` (สตริง, ไม่จำเป็น): `"publish"` (ค่าเริ่มต้น) หรือ `"list"` — แสดงรายการ artifacts ที่เผยแพร่ของผู้ใช้ (ชื่อเรื่อง, URL, อัปเดตล่าสุด), ไม่บังคับ ด้วย `limit`
- `force` (บูลีน, ไม่จำเป็น): เขียนทับโดยไม่มีการตรวจสอบความขัดแย้ง เฉพาะหลังจาก 409 จากการเขียนพร้อมกัน, หลังจากประนีประนวมแล้ว

## หมายเหตุ

- **เฉพาะการทำงานอิสระ** CSP ที่เข้มงวดบล็อกคำขอไปยังโฮสต์ภายนอกใด ๆ — สคริปต์ CDN, สไตล์ชีตภายนอก, รูปภาพระยะไกล, fetch/WebSockets. inline ทุก CSS/JS และฝัง assets เป็น `data:` URI
- **Responsive และตระหนักรู้เกี่ยวกับธีม** หน้าแสดงผลในธีมสว่างหรือมืดของผู้ดู; สไตล์ทั้งสอง (`prefers-color-scheme` บวก override `data-theme` ของผู้ดู) เนื้อหากว้าง scroll ภายในคอนเทนเนอร์ของมัน — body ของหน้าจะไม่ scroll แนวนอน
- **การอัปเดตข้ามเซสชันต้องการ `url`** การ redeploy พาธไฟล์เดียวกันจะนำ URL กลับมาใช้ใหม่เฉพาะภายในเซสชันที่เผยแพร่มัน; เพื่อให้ link ของ artifact เก่า, ค้นหา URL ด้วย `action: "list"` และส่งเป็น `url`
- **การเผยแพร่หันหน้าออกด้านนอก** เนื้อหาที่ส่งไปยังบริการ artifact อาจถูก cache แม้ว่าลบหลัง — อย่าเผยแพร่สิ่งใดก็ตามที่ต้องอยู่ในเครื่องส่วนตัว
- **อ่านกลับด้วย WebFetch** URL artifact ของ claude.ai สามารถ fetch ผ่าน WebFetch (ไม่ curl ซึ่งได้รับ app shell)
