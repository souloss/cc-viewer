# Artifact

HTML 또는 Markdown 파일을 Artifact로 렌더링합니다 — claude.ai에 호스팅되는 기본 비공개 웹페이지로, 사용자가 브라우저에서 열고 나중에 공유할 수 있습니다. 시각적 표현이 터미널 텍스트보다 나을 때 사용합니다.

## 사용 시점

- 시각적 산출물 발행: 보고서, 대시보드, 버그 조사 작성, UI 목업
- 이전에 발행된 페이지를 제자리에서 업데이트 (동일한 파일 경로는 동일한 URL로 재배포됨)
- 사용자의 기존 artifacts를 나열하여 이전 세션에서의 파일 찾기 (`action: "list"`)
- **하지 않을 때**: 로컬에 남겨둬야 할 콘텐츠, 일반 텍스트 답변, 보기 시 외부 네트워크 리소스가 필요한 모든 것——엄격한 CSP가 모든 외부 호스트를 차단합니다

## 매개변수

- `file_path` (문자열): 렌더링할 `.html` 또는 `.md` 파일의 경로. 파일은 발행 시 문서 틀로 래핑되므로 페이지 콘텐츠를 직접 작성합니다 — `<!DOCTYPE>`, `<html>`, `<head>` 또는 `<body>` 태그는 포함하지 않습니다. 동일 경로 → 재배포 시 동일 URL; 다른 경로는 새로운 URL을 청구합니다.
- `favicon` (문자열, 발행 필수): 브라우저 탭 아이콘으로 사용할 하나 또는 두 개의 emoji (예: `"📊"`). Emoji만, 마크업 없음. 재배포 시 동일하게 유지합니다 — 사용자는 아이콘으로 탭을 찾습니다.
- `description` (문자열): artifact 갤러리 카드에 표시되는 한 문장 부제목.
- `url` (문자열, 선택사항): 기존 artifact의 URL을 전달하여 이를 발행하지 않은 대화에서 제자리에 업데이트합니다. 없으면 새 대화는 항상 새 URL을 생성합니다.
- `label` (문자열, 선택사항): 버전 선택기에 표시되는 짧은 읽기 쉬운 버전 이름 (최대 60자).
- `action` (문자열, 선택사항): `"publish"` (기본값) 또는 `"list"` — 사용자의 발행된 artifacts 나열 (제목, URL, 마지막 업데이트), 선택적으로 `limit`.
- `force` (불리언, 선택사항): 충돌 검사 없이 덮어씁니다. 동시 쓰기에서 409가 나온 후, 협조된 경우에만 사용합니다.

## 참고사항

- **자체 포함만 가능합니다.** 엄격한 CSP는 모든 외부 호스트에 대한 요청을 차단합니다 — CDN 스크립트, 외부 스타일시트, 원격 이미지, fetch/WebSocket. 모든 CSS/JS를 인라인하고 자산을 `data:` URI로 포함합니다.
- **응답형이고 테마 인식입니다.** 페이지는 뷰어의 밝은 또는 어두운 테마로 렌더링됩니다; 둘 다 스타일을 지정합니다 (`prefers-color-scheme` 플러스 뷰어의 `data-theme` 재정의). 넓은 콘텐츠는 자체 컨테이너 내에서 스크롤됩니다 — 페이지 본문은 절대로 수평 스크롤하면 안 됩니다.
- **대화 간 업데이트에는 `url`이 필요합니다.** 이를 발행한 대화 내에서 동일한 파일 경로를 재배포하면 해당 대화 내에서만 URL을 재사용합니다; 이전 artifact의 링크를 유지하려면 `action: "list"`로 URL을 찾고 `url`로 전달합니다.
- **발행은 외부 지향입니다.** artifact 서비스로 전송된 콘텐츠는 나중에 삭제되어도 캐시될 수 있습니다 — 머신에서 비공개로 유지해야 하는 콘텐츠를 발행하지 마세요.
- **WebFetch로 읽기 돌려받기.** claude.ai artifact URL은 WebFetch를 통해 가져올 수 있습니다 (curl이 아님, curl은 앱 셸을 가져옵니다).
