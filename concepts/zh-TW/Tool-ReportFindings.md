# ReportFindings

將程式碼審查發現報告為類型化、結構化清單，由宿主 UI 原生呈現——而不是將其列印為聊天文字。

## 使用時機

- 結束程式碼審查，其活動指令明確說明使用此工具報告發現
- 套用修復後重新報告，當審查的套用指令要求此操作時（每個發現隨後攜帶 `outcome`）
- **不適用於**臨時意見、普通答案或審查指令指定不同輸出格式的情況——絕不與同一發現的文字副本一起使用

## 參數

- `findings`（陣列，必填，最多 32 個）：已驗證的發現，按最嚴重優先排序——如果沒有發現通過驗證，則為空陣列。每個發現：
  - `file`（字串，必填）：相對於倉庫的路徑。
  - `line`（數字，可選）：1 索引錨點行。
  - `summary`（字串，必填）：缺陷的單句陳述。
  - `failure_scenario`（字串，必填）：具體輸入/狀態 → 錯誤輸出或當機。
  - `category`（字串，可選）：短 kebab-case slug，例如 `correctness`、`simplification`、`efficiency`、`test-coverage`。
  - `verdict`（字串，可選）：`CONFIRMED` 或 `PLAUSIBLE`——驗證通過時設定；內連審查中不存在。
  - `outcome`（字串，可選）：僅在修復後重新報告時——`fixed`、`skipped` 或 `no_change_needed`。
- `level`（字串，可選）：審查執行時的工作量級別——`low`、`medium`、`high`、`xhigh` 或 `max`。

## 注意事項

- **呼叫一次。** 一次呼叫包含完整、已驗證、按嚴重程度排序的清單——而不是每個發現呼叫一次。
- **空是有效結果。** 如果沒有發現通過驗證，報告空陣列而不是用弱發現補充。
- **不要在文字中重複。** 當此工具報告結果時，發現不得也列印為聊天訊息。
- **`outcome` 僅用於重新報告。** 在第一次報告時不設定它；套用通過後，設定每個發現實際發生的情況。
