# Artifact

Перетворіть HTML або Markdown файл на артефакт — веб-сторінку з приватністю за замовчуванням, розміщену на claude.ai, яку користувач може відкрити в браузері та пізніше поділитися нею. Використовуйте це, коли розповідь через зображення краща за текст у терміналі.

## Коли використовувати

- Публікація візуального результату: звіту, інформаційної панелі, аналізу помилки або макета користувацького інтерфейсу
- Оновлення раніше опублікованої сторінки на місці (той же шлях файлу повторно розгортується до тієї ж URL)
- Отримання списку існуючих артефактів користувача для пошуку одного з попередньої сесії (`action: "list"`)
- **Не** для контенту, який повинен залишатися локальним, простих текстових відповідей або будь-чого, що вимагає зовнішніх мережевих ресурсів під час перегляду — суворий CSP блокує кожен зовнішній хост

## Параметри

- `file_path` (рядок): Шлях до файлу `.html` або `.md` для відтворення. Файл обертається в структуру документа під час публікації, тому пишіть вміст сторінки безпосередньо — без тегів `<!DOCTYPE>`, `<html>`, `<head>` або `<body>`. Той же шлях → та ж URL при повторному розгортанні; інший шлях отримує нову URL.
- `favicon` (рядок, необхідний для публікації): Один або два емодзі, використовані як значок вкладки браузера (наприклад, `"📊"`). Тільки емодзі, без розмітки. Вибирайте той же значок при повторних розгортаннях — користувачі знаходять вкладку за піктограмою.
- `description` (рядок): Однорядкове резюме, показане на картці галереї артефактів.
- `url` (рядок, необов'язковий): Передайте URL існуючого артефакту для оновлення його на місце з розмови, яка його не публікувала. Без неї нова розмова завжди створює нову URL.
- `label` (рядок, необов'язковий): Коротке, зрозуміле ім'я версії (макс. 60 символів), показане в засобі вибору версії.
- `action` (рядок, необов'язковий): `"publish"` (за замовчуванням) або `"list"` — список опублікованих артефактів користувача (назва, URL, час останнього оновлення), опціонально з `limit`.
- `force` (логічне значення, необов'язковий): Перезапис без перевірки конфлікту. Тільки після одержання 409 від одночасного запису після узгодження.

## Примітки

- **Тільки автономний контент.** Суворий CSP блокує запити до будь-яких зовнішніх хостів — скрипти CDN, зовнішні таблиці стилів, віддалені зображення, fetch/WebSockets. Вбудовуйте весь CSS/JS і вбудовуйте ресурси як URI `data:`.
- **Адаптивний та той, що підтримує тему.** Сторінки розташуються у світлій або темній темі глядача; оформляйте обидві (`prefers-color-scheme` плюс переопис `data-theme` глядача). Широкий вміст прокручується в межах власного контейнера — тіло сторінки ніколи не повинно прокручуватись горизонтально.
- **Оновлення в розмовах потребує `url`.** Повторне розгортання тієї ж сторінки повторно використовує URL лише в розмові, яка її опублікувала; щоб зберегти посилання на старіший артефакт, знайдіть його URL за допомогою `action: "list"` і передайте його як `url`.
- **Публікація є зовнішньою операцією.** Контент, відправлений на сервіс артефактів, може бути кешований, навіть якщо пізніше видалений — не публікуйте нічого, що повинно залишатися приватним на машині.
- **Прочитайте назад за допомогою WebFetch.** URL артефактів claude.ai можна отримувати за допомогою WebFetch (не curl, який отримує оболонку програми).
