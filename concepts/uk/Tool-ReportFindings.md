# ReportFindings

Повідомляйте про висновки перевірки коду як типізований, структурований список, який хост-інтерфейс розташовує нативно — замість друкування як текст у чаті.

## Коли використовувати

- Завершення перевірки коду, чиї активні інструкції явно говорять повідомляти про висновки цим інструментом
- Повторне повідомлення після застосування виправлень, коли інструкції перевірки просять це (кожен висновок потім несе `outcome`)
- **Не** для дотичних думок, звичайних відповідей або перевірок, чиї інструкції визначають інший формат виходу — і ніколи поряд з текстовим дублюванням тих самих висновків

## Параметри

- `findings` (масив, обов'язковий, макс. 32): Перевірені висновки, ранжовані від найсерйознішого — порожній масив, якщо нічого не витримало перевірку. Кожен висновок:
  - `file` (рядок, обов'язковий): Шлях відносно репозиторію.
  - `line` (число, необов'язковий): 1-індексована анкорна лінія.
  - `summary` (рядок, обов'язковий): Однорядкове твердження про недолік.
  - `failure_scenario` (рядок, обов'язковий): Конкретні входи/стан → неправильний вихід або крах.
  - `category` (рядок, необов'язковий): Коротка затримка формату кебаб-регістру, наприклад `correctness`, `simplification`, `efficiency`, `test-coverage`.
  - `verdict` (рядок, необов'язковий): `CONFIRMED` або `PLAUSIBLE` — встановлено, коли пройшла перевірка; відсутній при вбудованих переглядах.
  - `outcome` (рядок, необов'язковий): ТІЛЬКИ при повторному повідомленні після виправлень — `fixed`, `skipped` або `no_change_needed`.
- `level` (рядок, необов'язковий): Рівень зусилля, при якому проводилася перевірка — `low`, `medium`, `high`, `xhigh` або `max`.

## Примітки

- **Викличте один раз.** Один виклик з повним, перевіреним, ранжованим за серйозністю списком — не один виклик на висновок.
- **Порожнечина — дійсний результат.** Якщо нічого не витримало перевірку, повідомте порожній масив замість заповнення слабкими висновками.
- **Не дублюйте в тексті.** Коли цей інструмент повідомляє результати, висновки не повинні також друкуватися як повідомлення в чаті.
- **`outcome` тільки для повторних повідомлень.** При першому повідомленні залиште його невстановленим; після проходження застосування встановіть, що насправді трапилось з кожним висновком.
