# Artifact

HTML または Markdown ファイルを Artifact として レンダリングします — claude.ai で ホストされるデフォルト非公開のウェブページで、ユーザーはブラウザーで開いた後に共有することができます。視覚的なコミュニケーションがターミナルテキストに勝る場合に使用します。

## 使用タイミング

- ビジュアル成果物を公開: レポート、ダッシュボード、バグ調査レポート、または UI モックアップ
- 以前に公開したページをその場で更新 (同じファイルパスは再デプロイ時に同じ URL に展開される)
- ユーザーの既存の artifacts を一覧表示して、以前のセッションのファイルを検索 (`action: "list"`)
- **使用しない場合**: ローカルに保つ必要があるコンテンツ、プレーンテキストの回答、表示時に外部ネットワークリソースが必要なもの — 厳格な CSP はすべての外部ホストをブロックします

## パラメータ

- `file_path`（文字列）: レンダリングするための `.html` または `.md` ファイルのパス。ファイルは公開時にドキュメントスケルトンでラップされるため、ページコンテンツを直接記述します — `<!DOCTYPE>`、`<html>`、`<head>` または `<body>` タグは含めません。同じパス → 再デプロイ時に同じ URL；別のパスは新しい URL を要求します。
- `favicon`（文字列、公開に必須）: ブラウザータブアイコンとして使用する 1 つまたは 2 つの emoji (例: `"📊"`)。Emoji のみ、マークアップなし。再デプロイ時に同じに保ちます — ユーザーはアイコンでタブを見つけます。
- `description`（文字列）: artifact ギャラリーカードに表示される 1 文の小見出し。
- `url`（文字列、オプション）: 既存の artifact の URL を渡して、それを発行していない会話からその場で更新します。これがない場合、新しい会話は常に新しい URL を取得します。
- `label`（文字列、オプション）: バージョンピッカーに表示される短い読みやすいバージョン名 (最大 60 文字)。
- `action`（文字列、オプション）: `"publish"`（デフォルト）または `"list"` — ユーザーの公開済み artifacts を列挙 (タイトル、URL、最後の更新時刻)、オプションで `limit`。
- `force`（ブール値、オプション）: 競合チェックなしで上書きします。同時書き込みからの 409 の後、調整済みの場合のみ。

## 注意事項

- **自己完結のみ。** 厳格な CSP はあらゆる外部ホストへのリクエストをブロックします — CDN スクリプト、外部スタイルシート、リモート画像、fetch/WebSocket。すべての CSS/JS をインラインにして、アセットを `data:` URI として埋め込みます。
- **レスポンシブでテーマ認識。** ページはビューアーの明るいまたは暗いテーマでレンダリングされます；両方にスタイルを設定します (`prefers-color-scheme` プラスビューアーの `data-theme` オーバーライド)。広いコンテンツはその独自のコンテナー内でスクロールします — ページ本体は絶対に水平方向にスクロールしてはいけません。
- **会話間の更新には `url` が必要。** それを公開した会話内で同じファイルパスを再デプロイすると、その会話内でのみ URL を再利用します；古い artifact のリンクを保つには、`action: "list"` で URL を見つけ、`url` として渡します。
- **公開は対外向け。** artifact サービスに送信されたコンテンツは、後で削除されても キャッシュされる可能性があります — マシン上で非公開のままである必要があるコンテンツを公開しないでください。
- **WebFetch で読み戻し。** claude.ai artifact URL は WebFetch を通じてフェッチ可能です (curl ではなく、curl はアプリシェルを取得します)。
